## 2. 测试规范

### 2.1 命名规范
- 测试文件：`test_*.py`
- 测试类：`Test*`
- 测试方法：`test_*`
- 测试名称：描述具体行为，避免模糊

### 2.2 编写原则
- **一个测试一个行为**：测试名称不含"和"
- **使用真实代码**：尽量不使用 mock
- **清晰的断言**：预期结果明确

### 2.3 测试执行顺序
1. 运行单个测试文件：`pytest tests/unit/test_models.py -v`
2. 运行所有单元测试：`pytest tests/unit/ -v`
3. 运行所有测试：`pytest tests/ -v`

## 5. 测试运行命令

```bash
# 安装测试依赖
pip install pytest pytest-cov

# 运行所有测试
pytest tests/ -v

# 运行单元测试
pytest tests/unit/ -v

# 运行集成测试
pytest tests/integration/ -v

# 生成覆盖率报告
pytest tests/ --cov=app --cov-report=html

# 生成 JUnit 报告
pytest tests/ --junitxml=tests/reports/test_results.xml
```
